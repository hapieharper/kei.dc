# EDA 학습 문서(비즈니스 관점)

- 데이터베이스: `data/db/stt_sign_pple_mm.sqlite`
- 테이블: `stt_sign_pple_mm`
- 예시 칼럼: `결정금액(Integer>Currency)`

## 시작 전에: 이 분석을 왜 하나?

초보자 관점에서 EDA 목적은 복잡하지 않습니다.

1. 값이 정상적으로 들어왔는지 확인
2. 어디에 값이 몰려 있는지 확인
3. 너무 큰 값/이상한 값을 먼저 찾아 위험 신호 확인

즉, "바로 결론 내기"가 아니라 "데이터 상태를 파악"하는 단계입니다.

## 1. 비즈니스 질문 정의

`결정금액`은 심사 후 실제 지급이 확정된 금액입니다.  
따라서 지급 규모 관리, 이상지급 탐지, 담보별 손해구조 파악에 직접 연결됩니다.

핵심 질문:
1. 일반적인 지급금액 수준(중앙값)은 얼마인가?
2. 고액 지급(이상치)은 얼마나 자주 발생하는가?
3. 고액 지급은 어떤 담보에서 많이 발생하는가?

## 2. 전처리: 통화 문자열을 숫자로 변환

원본은 `₩300,000` 형태(TEXT)이므로 숫자형으로 변환 후 분석합니다.

```sql
SELECT
  CAST(
    NULLIF(
      REPLACE(REPLACE(REPLACE(COALESCE("결정금액(Integer>Currency)", ''), char(8361), ''), ',', ''), ' ', ''),
      ''
    ) AS REAL
  ) AS decided_amt
FROM "stt_sign_pple_mm";
```

## 5. 이상치 분석(IQR 방식)

- IQR = `Q3 - Q1 = 260,000`
- 상한선(Upper bound) = `Q3 + 1.5*IQR = 690,000`
- 이상치 건수 = `55,669`
- 이상치 비율 = `6.77%`

```sql
SELECT COUNT(*) AS outlier_cnt
FROM "stt_sign_pple_mm"
WHERE CAST(
        NULLIF(
          REPLACE(REPLACE(REPLACE(COALESCE("결정금액(Integer>Currency)", ''), char(8361), ''), ',', ''), ' ', ''),
          ''
        ) AS REAL
      ) > 690000;
```
