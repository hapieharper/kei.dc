# 신입 시행 점검 로그 (DBeaver 전용)

- 점검일: 2026-03-01
- 점검자 관점: "Python/노트북 없이 DBeaver만 사용"
- 점검 범위: `1주차/README.md`, `1주차/02_SQL실습/SQL_실행환경_가이드.md`, `1주차/02_SQL실습/주요질문_SQL.md`

## 점검 방식

1. README의 Day 1~4 순서를 그대로 따라가며 필요한 행동을 체크
2. Day 2(SQL 실습)에서 DBeaver만 사용한다고 가정하고 수행 절차 검토
3. 신입이 막힐 수 있는 지점과 혼동 포인트 기록

## 발견된 문제점

### 1) Day 2 경로에서 실행 방식이 3개(Python/노트북/DBeaver)로 섞여 초점이 흐려짐

- 심각도: 높음
- 신입 체감: "무엇이 필수인지 모르겠다"
- 상세:
1. README Day 2는 Python 스크립트와 노트북을 먼저 보여줌
2. DBeaver는 `SQL_실행환경_가이드.md` 내부에 후순위로 존재
3. DBeaver만 쓰려는 신입 입장에서 학습 경로가 분기되어 혼란 발생

### 2) SQL 실행환경 가이드의 준비물에 Python이 필수처럼 보임

- 심각도: 높음
- 신입 체감: "DBeaver만 쓸 건데 Python 설치가 꼭 필요한가?"
- 상세:
1. 준비물 1번이 `Python 3.10+`
2. DBeaver-only 경로에서는 Python이 필수 아님
3. 불필요한 설치 오해 유발 가능

### 3) DBeaver 절차가 짧아 실제 첫 접속 시 디테일이 부족함

- 심각도: 중간
- 신입 체감: "버튼은 눌렀는데 다음에 뭘 확인해야 할지 모르겠다"
- 상세:
1. SQLite 드라이버 최초 다운로드 팝업 대응이 없음
2. `Ctrl+Enter`(선택 쿼리 실행)과 전체 스크립트 실행 차이 설명 없음
3. 결과 저장 시 CSV Export 옵션(헤더 포함/인코딩) 가이드 없음

### 4) 결과 비교 기준이 문서에 명확하지 않음

- 심각도: 중간
- 신입 체감: "내 결과가 맞는지 어떻게 판단하지?"
- 상세:
1. `주요질문_SQL_샘플결과.md` 비교를 언급하지만
2. 정확히 어떤 쿼리 결과를 어디까지 같아야 정상인지 기준이 부족
3. 동일 쿼리라도 정렬/필터 차이로 미세 차이가 날 수 있다는 안내 없음

### 5) `results` 폴더에 여러 실행 결과가 누적되어 초보에게 노이즈가 됨

- 심각도: 낮음
- 신입 체감: "어떤 결과가 내가 방금 만든 건지 헷갈린다"
- 상세:
1. `run_20260301_210606`, `run_20260301_210619`, `run_20260301_212350` 등 다수 존재
2. 최신 결과를 식별하는 규칙 설명이 README에 없음

## 개선 제안 (우선순위)

### P1. DBeaver 전용 빠른 시작 섹션 분리

1. README Day 2 상단에 `DBeaver 전용 3단계` 박스 추가
2. Python/노트북은 "선택 경로"로 아래로 이동

### P1. SQL 실행환경 가이드 준비물 분리

1. 준비물 섹션을 `A. DBeaver 경로` / `B. Python 경로`로 분리
2. DBeaver 경로에는 Python 요구사항 제거

### P2. DBeaver 실행 디테일 추가

1. 첫 실행 드라이버 다운로드 팝업 안내
2. `Ctrl+Enter`(선택 실행) vs `Alt+X`(스크립트 전체 실행) 설명
3. CSV Export 권장 옵션(UTF-8, 헤더 포함) 명시

### P2. 정답 확인 규칙 추가

1. `주요질문_SQL.md` 각 쿼리에 "검증 포인트" 1줄 추가
2. 예: "상위 5행의 disease_code가 J209/J304 포함이면 정상"

### P3. 결과 폴더 운영 규칙 추가

1. README에 "가장 최근 run 폴더만 확인" 규칙 명시
2. 필요 시 이전 run 정리 방법 안내

## 결론

DBeaver-only 신입 기준으로는 현재 자료가 충분히 실행 가능하나,  
"필수 경로"와 "선택 경로"가 섞여 있어 첫 진입 혼란이 존재한다.  
README와 SQL 실행환경 가이드를 DBeaver 중심으로 재정렬하면 온보딩 속도가 확실히 빨라질 것으로 판단된다.
